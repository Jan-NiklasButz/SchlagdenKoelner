# Game Night UI - Interactive Scoreboard

## Purpose
Generate a responsive web-based UI for a 15-game tournament where each game awards points (1-15 points respectively). The interface tracks player scores in real-time and displays game suggestions with clickable integration.

## Context
- **Game Format**: 15 games total
  - Game 1: 1 point
  - Game 2: 2 points
  - ...
  - Game 15: 15 points
- **Players**: Multiple players competing (typically 2-4)
- **Tech Stack**: HTML, CSS, JavaScript (vanilla or with a lightweight framework)

## What to Generate

### UI Layout & Components
1. **Player Score Display**
   - Show each player's current total score
   - Display points earned in the current game
   - Clear visual feedback for scoring

2. **Charts & Analytics**
   - Pie chart showing overall points distribution between players
   - Progress bar showing how many games are completed
   - Optional: Bar chart showing individual game scores

3. **Game Selector & Launcher**
   - Display all 15 games in a clickable list/grid
   - Show point value for each game
   - Mark completed games visually
   - Ability to launch/start a game by clicking

4. **Game Suggestions**
   - Include suggestions for game types at each point level:
     - 1-3 points: Quick games (trivia questions, riddles, rock-paper-scissors)
     - 4-7 points: Medium games (word games, memory, drawing challenges)
     - 8-12 points: Complex games (strategy games, puzzles, skill challenges)
     - 13-15 points: Ultimate games (tournament finals, complex strategy)
   - Allow users to customize game types

### Features
- **Real-time score tracking** with clear feedback
- **Responsive design** that works on desktop and tablet
- **Dark or light theme** toggle
- **Simple game start mechanism** (click to launch, or expand for game instructions)
- **Clear visual hierarchy** - points and rankings are immediately obvious

## Input Requirements (User Provides)
- Player names (or use default: Player 1, Player 2, etc.)
- Optional: Custom game suggestions/types
- Optional: Theme preference

## Output Format
Return complete, functional HTML file with:
- Embedded CSS (in `<style>` tag) with modern, clean design
- Embedded JavaScript (in `<script>` tag) for interactivity
- No external dependencies unless essential (prefer vanilla JS or lightweight libraries like Chart.js for pie chart)
- Comments explaining the logic
- Ready to open in a browser immediately

## Code Style
Follow the workspace instructions:
- Use `snake_case` for all variables and functions
- Include detailed comments explaining the logic
- Document any libraries used with links to their documentation
- Keep code simple and readable for learning purposes

## Example Invocation
"Create the game night UI. Players are: Alice, Bob, and Charlie. Use these game suggestions for points 1-5: Trivia, Quick Draw, 20 Questions, Rhyme Time, Brain Teaser."

Generated UI should be immediately playable and pleasant to use during an actual game night.
