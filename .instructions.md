# Code Generation Instructions

You are helping someone new to programming. Prioritize clarity and consistency above all else.

## Variable & Function Naming

**Use `snake_case` for all variable and function names** (e.g., `user_name`, `calculate_total`, `data_count`).

- Never use camelCase or PascalCase
- Use descriptive names that explain the purpose (e.g., `student_count` instead of `sc`)
- Be consistent across the entire codebase

## Code Comments & Explanations

**Add detailed comments explaining WHAT the code does and WHY.**

- Start each function with a docstring explaining its purpose
- Comment non-obvious logic and complex operations
- Add comments above each major code section
- Explain why you chose a particular approach, not just what it does

**Example:**
```python
def calculate_average_score(scores_list):
    """
    Calculate the average score from a list of student scores.
    
    Args:
        scores_list: A list of numerical scores
    
    Returns:
        The average as a float, or 0 if list is empty
    """
    # Guard against empty lists to prevent division errors
    if not scores_list:
        return 0
    
    # Sum all scores and divide by count
    total_score = sum(scores_list)
    average = total_score / len(scores_list)
    
    return average
```

## Library Documentation Links

**When using a library for the first time, always include a comment with the documentation link.**

Example:
```python
# Installing requests: https://docs.python-requests.org/
import requests

# Using pandas: https://pandas.pydata.org/docs/
import pandas as pd
```

## Code Simplicity

- Avoid deep nesting; break complex logic into smaller functions
- Keep functions focused on one task
- Use clear, simple variable names that anyone can understand
- Explain the input and output of each function
